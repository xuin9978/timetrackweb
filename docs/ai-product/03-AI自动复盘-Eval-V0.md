# AI 自动复盘 Eval V0

## 1. Eval 目标

评估 AI 自动复盘是否基于真实时间记录生成稳定、准确、可读、可执行的日复盘，重点防止编造事实和输出结构漂移。

## 3. 评分维度

| 维度 | 说明 | 分值 |
| --- | --- | --- |
| 事实一致性 | 是否只使用输入中存在的事件、标签、日期、时长 | 1-5 |
| 结构完整性 | 是否包含约定输出模块 | 1-5 |
| 数据解释能力 | 是否能正确解释时间分配、占比和异常 | 1-5 |
| 建议可执行性 | 建议是否具体、轻量、与当天数据相关 | 1-5 |
| 克制程度 | 是否避免过度推断目标、情绪、原因和偏好 | 1-5 |
| 语言可读性 | 中文表达是否清晰、自然、便于快速阅读 | 1-5 |
| 兜底质量 | 数据不足或异常时是否给出合适提示 | 1-5 |

## 4. Rubric

### 5 分

- 完全基于输入数据。
- 输出结构完整。
- 能准确引用关键时间分配和标签。
- 建议具体且不过度承诺。
- 无编造、无明显废话。

### 4 分

- 基本基于输入数据。
- 结构完整或仅有轻微格式偏差。
- 解释和建议大体合理。
- 有少量泛化表述，但不影响理解。

### 3 分

- 主要结论可用。
- 有部分笼统表达或弱相关建议。
- 存在轻微结构缺失或数据引用不清。

### 2 分

- 输出可读但事实依据弱。
- 有明显泛化、过度推断或结构缺失。
- 需要重写 Prompt 或输出约束。

### 1 分

- 编造事件、标签、目标、情绪或时长。
- 输出结构严重不符合要求。
- 对用户产生明显误导。

## 5. 通过标准

| 项目 | 标准 |
| --- | --- |
| 单样本通过 | 总体评分 >= 4，且事实一致性 >= 4 |
| 批量通过 | 80% 以上样本单样本通过 |
| 硬性失败 | 任何样本出现严重编造事实，必须记录 badcase |
| 上线前门槛 | 正常、少数据、无数据、标签缺失四类样本必须通过 |
