# 一天之旅 Eval 运行记录

## 0. 文件定位

本文件用于记录「一天之旅」AI 生成功能的 Golden Set 回归测试。

Eval 的目标不是证明接口能不能返回内容，而是检查生成结果是否能稳定对齐 Golden Reference，包括时间事实、关键词粒度、早中晚分段、阶段归纳和一天回顾。

### 2.4 Golden 对比结论

#### 2026-06-01

未通过 Golden 对比。

主要问题：

- 关键词过泛：生成版为 `阅读、复盘、日常、营养补剂、图书馆`，缺少 Golden 中的 `周复盘、月度复盘、6 月规划、历史复盘整理、Learn Claude Code` 等主线粒度。
- 分段边界不够贴近 Golden：生成版把 `11:30` 午饭和 `12:00` 月度复盘放入早上；Golden 将 11:30 之后归入下午。
- 阶段归纳过粗：`◆ 阅读和复盘`、`◆ 复盘和日常`、`◆ 复盘、日常和图书馆` 不能覆盖具体事实。
- 一天回顾偏短：遗漏黑芝麻粉、亚麻籽粉、刷牙洗漱、5 月份复盘语音输入、1-3 月份回顾、中途 7 分钟吃饭、2024/2025 文档合并等关键事实。

已通过：

- 标题日期正确。
- 时间点没有漂移，输出中的时间点均来自输入 payload。
- 没有发现明显虚构内容。

#### 2026-06-02

未通过 Golden 对比。

主要问题：

- 结构失败：生成结果缺少 `### 一天回顾`，也没有 ☀️ / 🌇 / 🌃 三段第一人称回顾。
- 关键词不完整：生成版漏掉 `Learn Claude Code`、`6 月规划`、`Agent 专项归档`、`Trae` 等 Golden 主线词。
- 阶段归纳过泛且有事实表达问题：早上 `◆` 写入 `老爸生日`，但原始事实是“帮老爸弄照片”，不是“老爸生日”。
- 下午和晚上归纳过粗：`日常活动、健身、美食、社交` 过泛，无法覆盖吃羊肉串、健身到 8:40、野人先生冰淇淋、走路回家、装三重镁、聊天等事实。

已通过：

- 标题日期正确。
- 早上 / 下午 / 晚上主体结构存在。
- 时间点没有漂移，输出中的时间点均来自输入 payload。

## 4. 本轮结论

- 6 月 1 日和 6 月 2 日接口均成功调用，说明链路可用。
- 两个样例都未通过 Golden 对比，说明当前 V1.2 规则对未注入 Golden Reference 的日期泛化不够稳定。
- 时间点漂移问题在本轮未复现，`displayStartTime` 机制有效。
- 下一步不建议直接进入复制、下载、保存功能；建议先做 V1.3 Prompt / 输入构造修复，重点解决 `◆` 归纳过粗、一天回顾缺失 / 偏短、午饭后分段和跨样例事实迁移问题。
