# 一天之旅 AI 生成功能文档资产

本目录用于存放「一天之旅」AI 生成功能的内部文档资产，包括系统提示词、最高优先级输出范本、few-shot 样例、测试样例、提示词迭代记录和作品集证据。

本目录不是前端组件目录，也不是用户可见页面目录。运行时代码、组件实现和 AI 接口不应直接混入本目录。

## 文件说明

| 文件 / 目录 | 作用 |
| --- | --- |
| `一天之旅系统提示词.md` | 后续后台 LLM 调用使用的系统提示词设计文档，用于约束 AI 严格按固定格式生成「一天之旅」。 |
| `03_Golden_一天之旅最终范本.md` | 最高优先级输出模板。当前以 2026 年 6 月 4 日最终版为格式标准。 |
| `examples/` | 保存 4 个原始 md 样例，作为 few-shot 样例、测试样例和作品集证据。 |
| `02_Prompt_一天之旅提示词迭代记录.md` | 记录「一天之旅」输出模板约 24 轮迭代形成的过程，用于后续 Prompt 设计、Eval 和作品集表达。 |

## 样例优先级

1. `03_Golden_一天之旅最终范本.md`：最高优先级格式标准。
2. `examples/2026年6月4日时间记录与一天之旅.md`：包含原始时间记录和最终版输出，是最高优先级测试样例。
3. `examples/2026年6月1日原始日历与一天之旅.md`、`examples/2026年6月2日原始日历与一天之旅.md`、`examples/2026年6月3日原始日历与一天之旅.md`：补充 few-shot 样例和边界参考。

## 使用边界

- 这些文件可用于提示词设计、few-shot 参考、测试样例、Eval 标准和作品集证据。
- 不应把 examples 下的原始样例改写成总结版。
- 不应把未来 AI 功能写成本目录已经实现的运行能力。
- 后续如新增真实 AI 接口，应在代码中读取或内置经过确认的 Prompt 版本，并在本目录记录对应版本。
