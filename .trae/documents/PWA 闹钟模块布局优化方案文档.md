# PWA 闹钟模块布局优化方案 (适配 iPhone 15 Pro Max)

本方案旨在解决 PWA 模式下闹钟模块底部留白过多问题，通过调整 CSS 布局实现全屏卡片效果。**特别强调：所有修改严格限定在移动端/PWA 模式下，绝对不改变桌面端网页浏览器的任何布局排版。**

## 1. 核心目标
*   **消除留白**：移除 PWA 模式下闹钟卡片下方的灰色空白区域。
*   **全屏适配**：让卡片高度自适应填满剩余空间，直达底部导航栏上方。
*   **视觉一致性**：复刻网页版的大字体、居中布局和淡绿色按钮风格。
*   **零干扰原则**：通过 Tailwind 的 `md:` 前缀和 CSS 媒体查询，确保桌面端布局代码 100% 隔离，保持原样。

## 2. 技术实施细节

### A. 容器布局调整 (`components/Alarm.tsx`)

*   **策略**：利用 Tailwind 的响应式前缀区分移动端和桌面端样式。
*   **代码变更**：
    *   **移动端 (默认)**：`h-full w-full bg-white rounded-[32px] flex flex-col items-center justify-center` (全屏白卡片)。
    *   **桌面端 (`md:`前缀)**：`md:h-[85vh] md:max-w-2xl md:bg-white/80` (保持原有的玻璃卡片样式)。
    *   **实现方式**：在 `Alarm.tsx` 中，针对容器 `div` 使用条件类名，确保移动端样式不污染桌面端。

### B. 父级容器调整 (`App.tsx`)

*   **策略**：仅调整移动端的 Padding，桌面端 Padding 保持不变。
*   **代码变更**：
    *   **移动端**：`p-4` (保留四周间隙，实现悬浮感)。
    *   **桌面端**：`md:p-6` (保持原有间距)。
    *   **Flex 布局**：确保父容器在移动端也是 `flex-col` 且 `h-full`，以便子元素能撑开高度。

### C. 内容排版微调

*   **时间显示**：
    *   移动端：`text-7xl font-normal`。
    *   桌面端：`md:text-9xl md:font-light` (保持原样)。
*   **启动按钮**：
    *   统一更新为 `bg-[#E0F2E9] text-[#2E7D32]` (淡绿底深绿字)，此样式优化通用于双端，提升视觉体验。
*   **模式切换器**：
    *   移动端：调整 `gap` 或 `margin` 使其布局更舒展。
    *   桌面端：保持原有紧凑布局。

## 3. 预期效果

*   **PWA 端 (iPhone 15 Pro Max)**：用户将看到一个悬浮在灰色背景上的、几乎填满屏幕的白色大圆角卡片，底部无大片死角，视觉重心完美居中。
*   **网页浏览器端**：**完全无变化**。布局、尺寸、玻璃质感均与修改前保持 1:1 一致。

## 4. 执行计划

1.  **修改 `Alarm.tsx`**：使用响应式类名重构容器，实现 PWA 全高卡片。
2.  **修改 `App.tsx`**：调整移动端父容器 Padding。
3.  **验证**：分别在移动端模拟器和桌面浏览器中检查，确保互不影响。
