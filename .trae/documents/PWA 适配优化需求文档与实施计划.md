# PWA 适配与移动端排版优化计划

## 1. 需求分析与 PWA 优化文档
根据您的描述与截图，当前移动端浏览时导航栏排版混乱。为了实现理想的“底部导航栏”效果，并修复 CSS 选择器失效的问题，我制定了以下规范：

### 移动端 PWA 适配需求文档
**1.1 触发条件**
- **响应式布局**：当屏幕宽度小于 768px（手机端）时，自动切换到底部导航模式，解决手机浏览器直接打开时的排版混乱。
- **PWA 增强**：在 PWA 模式（`display-mode: standalone`）下，额外适配底部安全区域（Home Indicator）。

**1.2 布局规范**
- **位置**：固定在屏幕底部（Fixed Bottom）。
- **容器样式**：白色背景、顶部圆角、阴影、高度自适应（包含安全区域）。
- **元素展示**：**隐藏所有中文标签**，仅显示图标。

**1.3 按钮排列顺序**
根据您的要求 `div`、`div`、`button`、`div`、`button`，调整后的视觉顺序为：
1. **闹钟** (左一)
2. **历史** (左二)
3. **添加** (中间，黑色圆形按钮)
4. **日历** (右二)
5. **设置** (右一)

---

## 2. 技术修复方案

### 2.1 修复 CSS 选择器失效问题
**问题根源**：`Sidebar.tsx` 中的 `GlassCard` 组件并未实际渲染 `GlassCard` 这个 CSS 类名，导致 `index.html` 中的 `.pwa-mode .App .GlassCard` 样式规则全部失效。
**修复**：在 `Sidebar.tsx` 中显式添加 `sidebar-container` 类名。

### 2.2 实施 CSS 样式重构 (`index.html`)
我们将重写 `<style>` 块，使用 `@media (max-width: 768px)` 媒体查询来覆盖所有移动端场景：

1.  **通用移动端适配**：
    - 将 Sidebar 固定到底部。
    - 使用 `order` 属性重排按钮顺序：
      - 闹钟: `order: 1`
      - 历史: `order: 2`
      - 添加: `order: 3`
      - 日历: `order: 4`
      - 设置容器: `order: 5`
    - 隐藏 `span` 文字标签。
2.  **PWA 特有适配**：
    - 针对 `.pwa-mode` 增加 `padding-bottom: env(safe-area-inset-bottom)`。

## 3. 执行步骤
1.  **修改 `Sidebar.tsx`**：添加识别类名。
2.  **修改 `index.html`**：重写 CSS 样式，实现上述布局与顺序。
3.  **验证**：重启服务器，提供预览链接供您在手机上验证。
