# 移动端网页体验优化计划

## 1. 响应式布局重构 (Calendar.tsx)

针对顶部被遮挡和布局挤压问题，进行以下修改：
- **重构头部导航栏**：移除 `absolute` 绝对定位，改用 Flexbox 垂直排列（移动端）/ 水平排列（桌面端）。这样可以避免标题被日期选择器遮挡，也能自然地顶出足够的空间。
- **优化高度控制**：将移动端的固定高度 `h-[85vh]` 改为 `h-full` 或 `min-h-[calc(100vh-120px)]`，确保内容区域能充分利用剩余空间且不溢出。
- **适配 PWA 安全区域**：为头部容器添加 `pt-[env(safe-area-inset-top)]`，防止刘海屏遮挡标题。

## 2. 侧边详情面板适配 (Calendar.tsx)

目前的详情面板 (`EventPanel`) 在移动端是侧边滑出 (`w-80`)，这在手机上会挤压日历或需要横向滚动。优化方案：
- **移动端底部抽屉模式**：在小屏幕下，将 `EventPanel` 改为覆盖在日历上方的底部抽屉（或模态层），或者直接在日历下方展示（根据剩余空间）。
- **交互优化**：增加显眼的关闭/收起按钮。

## 3. 全局样式微调 (index.html & App.tsx)

- **字体与排版**：
  - 调整根字体大小，确保移动端正文不小于 16px（防止 iOS 输入框自动缩放）。
  - 增加行高至 1.5-1.6。
- **点击区域**：
  - 确保所有按钮（特别是日期切换箭头、模式切换按钮）的 `padding` 足够，使其点击热区达到 48x48px。
- **App 容器优化**：
  - 移除移动端根容器的垂直居中 (`items-center`)，改为顶部对齐 (`items-start`)，避免内容过长时头部被切断。
  - 使用 `dvh` 单位（如有支持）或 JS 动态计算高度，解决移动浏览器地址栏遮挡问题。

## 4. 执行步骤

1.  **修改 `components/Calendar.tsx`**：重构 Header 布局，移除绝对定位，添加安全区域 Padding。
2.  **修改 `components/Calendar.tsx`**：调整 `EventPanel` 在移动端的显示逻辑（样式调整）。
3.  **修改 `App.tsx`**：调整根容器的 Flex 对齐方式。
4.  **修改 `index.html`**：添加全局 CSS 变量和排版优化样式。
5.  **验证**：重启服务器并验证布局。
