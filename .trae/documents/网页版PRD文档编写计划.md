# iOS 26 Liquid Calendar 网页版 PRD 文档

## 一、产品概述

### 1.1 产品定位

iOS 26 Liquid Calendar 是一款现代化的时间管理与日程规划应用,提供流畅的日历视图、精准的时间追踪和智能的数据管理功能。网页版作为跨平台解决方案,提供与iOS应用一致的用户体验。

### 1.2 核心价值

* **直观的日历管理**: 多视图切换(日/周/月),支持快速创建和编辑日程

* **精准时间追踪**: 秒表和倒计时双模式,支持自动记录到日历

* **灵活标签系统**: 自定义标签分类,支持拖拽排序和可视化过滤

* **云端同步**: 基于Supabase的数据存储,支持多设备同步

* **离线支持**: 本地缓存机制,离线状态下仍可使用

## 三、功能模块

### 3.1 日历模块 (Calendar)

#### 3.1.1 视图模式

* **日视图**: 单日时间网格,展示当日所有日程

* **周视图**: 7天时间网格,横向滚动

* **月视图**: 传统日历格子,显示每月日期

#### 3.1.2 核心功能

1. **日期导航**

   * 上一页/下一页切换

   * 快速回到今天

   * 点击日期跳转到日视图

2. **日程创建**

   * 点击日历单元格创建事件

   * 拖拽选择时间段创建事件

   * 快速添加按钮(智能计算默认时间)

3. **日程编辑**

   * 点击事件卡片打开编辑弹窗

   * 支持修改标题、时间、标签

   * 支持删除事件

4. **日程面板**

   * 右侧面板展示当前视图的所有事件

   * 支持三种显示模式:列表/分类/统计

   * 支持标签过滤

   * 支持按分类折叠/展开

#### 3.1.3 数据模型

```typescript
interface CalendarEvent {
  id: string;
  title: string;
  startTime: string;  // HH:mm格式
  endTime: string;    // HH:mm格式
  category: string;    // 关联标签ID
  date: Date;
  description?: string;
}
```

### 3.2 闹钟模块 (Alarm)

#### 3.2.1 模式

* **秒表模式**: 正向计时,记录活动时长

* **计时模式**: 倒计时,设置目标时间

#### 3.2.2 核心功能

1. **时间显示**

   * 大字体显示当前时间

   * 实时更新(秒级精度)

2. **控制按钮**

   * 启动:开始计时

   * 停止:暂停计时

   * 继续:恢复暂停的计时

   * 完成:结束并记录到日历

3. **记录会话**

   * 选择时间段关联的标签

   * 自动创建日程事件

   * 支持多个标签批量创建

#### 3.2.3 数据模型

```typescript
interface AlarmState {
  status: 'idle' | 'running' | 'paused';
  mode: 'stopwatch' | 'timer';
  startTime: number;
  accumulatedTime: number;
  totalDuration: number;
}
```

### 3.3 历史模块 (History)

#### 3.3.1 核心功能

1. **日期选择**

   * 迷你日历选择历史日期

   * 格式: yyyy年M月d日

2. **事件展示**

   * 按时间倒序排列

   * 支持分类视图切换

   * 显示事件时长统计

3. **标签管理**

   * 打开标签管理弹窗

   * 支持标签的增删改查

   * 支持拖拽排序

#### 3.3.2 视图模式

* **列表模式**: 按时间顺序展示

* **分类模式**: 按标签分组展示,显示总时长

### 3.4 标签系统

#### 3.4.1 标签属性

* 标签名称

* 颜色(预设Tailwind类)

* 图标(Emoji)

* 排序序号

#### 3.4.2 标签功能

1. **创建标签**

   * 自定义名称、颜色、图标

   * 支持快速创建

2. **编辑标签**

   * 修改标签属性

   * 支持删除标签

3. **标签排序**

   * 拖拽调整顺序

   * 自动保存到云端

4. **标签过滤**

   * 可视化复选框过滤

   * 控制日程面板显示

#### 3.4.3 数据模型

### 3.5 用户认证系统

#### 3.5.1 认证方式

#### 3.5.2 功能

4. **数据隔离**

   * 未登录显示模拟数据

### 3.6 设置与数据管理

#### 3.6.1 设置功能

1. **数据导入**

   * 支持ICS文件导入

   * 智能匹配标签

2. **数据导出**

   * 导出为ICS格式

   * 同步到系统日历

3. **数据恢复**

   * 从本地备份恢复

   * 从云端加载

4. **网络状态**

   * 实时显示在线/离线状态

   * 离线时使用本地缓存

## 五、数据存储

### 5.1 云端存储(Supabase)

#### 5.1.1 events表

```sql
CREATE TABLE events (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL REFERENCES auth.users,
  title TEXT NOT NULL,
  start_time TIMESTAMPTZ NOT NULL,
  end_time TIMESTAMPTZ NOT NULL,
  category TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

#### 5.1.2 tags表

```sql
CREATE TABLE tags (
  id TEXT PRIMARY KEY,
  user_id UUID NOT NULL REFERENCES auth.users,
  label TEXT NOT NULL,
  color TEXT NOT NULL,
  icon TEXT,
  "order" INTEGER DEFAULT 0,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

### 5.2 本地存储

* **LocalStorage**:

  * `events_cache_{userId}`: 事件缓存

  * `tag_order_guest`: 访客标签排序

  * `backup_events_{userId}`: 数据备份

## 六、性能优化

### 6.1 优化策略

1. **乐观更新**: 操作先更新UI,后台同步数据库
2. **数据分页**: 标签分页加载(每页50条)
3. **缓存机制**: 本地缓存事件数据,减少网络请求
4. **防重复导入**: 导入时检查重复事件
5. **智能加载**: 只加载当前视图可见日期范围的事件

### 6.2 错误处理

* **JWT过期**: 自动刷新会话,提示重新登录

* **网络错误**: 降级到本地缓存,显示离线提示

* **数据同步**: 失败时回滚乐观更新
