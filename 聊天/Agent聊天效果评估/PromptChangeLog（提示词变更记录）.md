# PromptChangeLog（提示词变更记录）

PromptChangeLog 记录每次 Agent 提示词为什么改、改了什么、预期改善什么、可能带来什么副作用。

## 2026-07-19：V1.1 自然对话优化

### 改前问题

系统提示词要求回答时必须区分“上下文事实、统计趋势、你的推断”。这有助于可信度，但会让普通聊天回答变成报告。

### 修改内容

- Agent 人设从“个人时间管理 Agent”改为“个人时间伙伴”。
- 增加“自然聊天”要求。
- 增加今日安排、复盘分析、情绪压力、行动写入四类回答策略。
- 将“每次都显式区分事实/统计/推断”调整为“复盘或询问依据时再展开依据”。
- 保留纯文本输出约束。
- 保留不编造、不越权、不诊断的边界。

### 预期改善

- 普通计划问题更自然。
- 日常对话不再直接变成报告。
- 情绪压力类回答先接住感受。
- 行动草案类回答更具体，但仍等待确认。

### 可能副作用

- 自然度提升后，回答可能减少显式依据说明。
- 需要通过来源提示和可展开依据在后续 V1.5 弥补透明度。
- 如果 prompt 太强调陪伴，可能导致回答过度拟人，需要 Badcase 持续观察。

### 复测结果

本轮复测覆盖三层：

一、prompt 结构测试：确认仍禁止 Markdown，且不再强制所有回答都机械展示事实、统计、推断。

二、GoldenSet synthetic 评估：用 40 条单轮样本检查自然度、边界感、可执行性和陪伴感。

三、多轮流程骨架检查：确认 8 组流程已覆盖上下文连续性、语气连续性和任务推进能力。

实际分数和失败标签以 `EvaluationLog.synthetic.latest（合成上下文最新评估）.md` 为准。

### 是否产生副作用

当前需要继续观察两个副作用：

一、回答变自然后，是否会过少说明依据。

二、日常陪伴感增强后，是否会出现过度拟人或越界安慰。

## 2026-07-19：真实评估后的下一轮 prompt 问题

### 改前问题

真实 synthetic 评估显示，当前 prompt 虽然让回答更自然，但没有明确区分“用户显式要求基于记录回答”和“用户只是泛化求助”。因此 Agent 会在一些泛化问题中直接套用背景日程和日记。

### 待修改方向

- 增加上下文意图门控：用户没有明确要求基于记录时，先给通用方法，再轻提示可结合记录细排。
- 对“日常对话 / 吐槽 / 不想听建议”类问题降低主动分析强度。
- 强化多轮和作品集复盘场景下的纯文本输出约束。

### 预期改善

- 减少上下文抢答。
- 减少用户没有授权时的过度分析。
- 降低多轮 Markdown 符号回归。

### 可能副作用

- Agent 可能变得更谨慎，回答速度和直接性下降。
- 如果门控过强，可能降低“主动理解用户时间上下文”的产品价值。

### 复测结果

待下一轮 prompt 修改后复测。
