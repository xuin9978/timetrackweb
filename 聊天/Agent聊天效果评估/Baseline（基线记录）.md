# Baseline（基线记录）

Baseline 用于记录每轮优化前 Agent 的真实表现，证明迭代不是凭感觉发生的。

## Baseline 001：V1 到 V1.1 前

时间：2026-07-19

触发背景：用户反馈聊天回答“不自然”，不像 DeepSeek 那样自然，更像时间管理报告。

当时表现：

- 能读取日历和日记上下文。
- 能给出计划和复盘建议。
- 但回答容易机械区分“上下文事实、统计趋势、推断”。
- 普通计划问题也容易写成报告。
- 日常对话和情绪类问题缺少自然承接。
- Markdown 符号曾经出现在回答中。

代表性问题：

一、帮我安排今天的优先级

优化前失败标签：报告腔、过度分析、Markdown 符号风险。

V1.1 期待结果：先一句自然回应，再给 2 到 4 个行动建议；不机械展示事实、统计、推断。

二、我最近是不是状态不太好？

优化前失败标签：缺少情绪承接、心理诊断化风险、过度分析。

V1.1 期待结果：先承接感受，再谨慎说明依据是否充分，最后给一个轻量调整建议。

三、帮我生成一个今天下午的安排草案

优化前失败标签：越权行动风险、计划过满。

V1.1 期待结果：明确这是草案，尚未写入日程，并留出缓冲。

已采取动作：

- 增加纯文本输出约束。
- 将 Agent 人设调整为“个人时间伙伴”。
- 增加问题类型回答策略。
- 快捷问题改为更自然的表达。

待验证：

- 40 条 GoldenSet 的真实输出评分。
- 8 组多轮流程里的语气连续性和任务推进能力。
- 日常对话是否不再被强行效率化。

修复后结果记录位置：

- 单轮结果：`EvaluationLog.synthetic.latest（合成上下文最新评估）.md`
- 多轮结果：`EvaluationLog.flows.synthetic.latest（多轮流程最新评估）.md`
- 失败样例：`Badcase（坏案例）.md`
