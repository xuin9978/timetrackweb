# 作品集叙事：从聊天功能到 Agent 体验评估机制

## 项目背景

TimeTrack 已经具备日历、日记、历史记录和标签系统。V1 Agent 的第一步，是让用户可以用自然语言询问今天、本周、近期状态和长期时间模式。它已经能够读取分层日历上下文和分层日记上下文，但真实使用中暴露出一个体验问题：回答有时太像分析报告，不像自然聊天。

这个问题不是简单的“prompt 写得不够好”，而是一个 Agent 体验质量如何被定义、评估、迭代的问题。因此，我建立了一套 Agent 体验评估与迭代机制。

## 方法

我把 Agent 体验拆成三层证据：

一、理想体验定义。通过 GoldenSet 和 Golden Answer 明确什么叫好回答。

二、真实表现记录。通过 Baseline 和 EvaluationLog 记录当前 Agent 的实际输出。

三、迭代闭环。通过 Badcase、PromptChangeLog 和 IterationLog 记录问题、修复和复测。

## 评估维度

这套机制不只看答案是否“正确”，还评估它是否像一个真正可用的个人时间 Agent：

- 自然度：像聊天，不像报告。
- 上下文准确性：基于日历和日记，不乱编。
- 边界感：不声称已经写入日程，不做心理诊断。
- 可执行性：建议具体，用户能马上行动。
- 陪伴感：能轻轻接住状态，但不过度煽情。

## 作品集价值

这套机制证明的不是“我会调 prompt”，而是：

我能发现 Agent 体验问题。

我能把模糊体验拆成可评估标准。

我能用 GoldenSet 和 Badcase 建立测试资产。

我能通过迭代日志证明优化过程。

我能兼顾作品集展示、工程回归和隐私边界。

## 当前版本

当前评估体系服务于 V1.1 自然对话优化。后续 V1.5、V2、V3 可以继续沿用这套机制，新增更多 GoldenSet、工具调用场景、记忆场景和确认写入场景。
